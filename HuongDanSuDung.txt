================================================================================
                    HUONG DAN SU DUNG - RAGAS Answer Filtering
================================================================================

Du an nay la ma Python (khong can bien dich). "Chay chuong trinh" co nghia
la kich hoat moi truong ao, roi chay notebook hoac script Python tuong ung.

Dieu kien tien quyet: da cai dat theo file HuongDanCaiDat.txt
(moi truong ao, pip install -r requirements.txt, file .env voi OPENAI_API_KEY
neu chay RAGAS / LLM-judge).

Pipeline:

  retrieve -> generate -> filter(context, answer) -> accept / reject

Phuong phap chinh: RAGAS-feature filter (notebook 3.1 -> 3.2).
Baseline: LLM-as-judge (notebook 4), DeBERTa / NLI (notebook 5 / script).

--------------------------------------------------------------------------------
1. KICH HOAT MOI TRUONG
--------------------------------------------------------------------------------

  Windows:
      venv\Scripts\activate

  Linux / macOS:
      source venv/bin/activate

  cd <duong-dan-toi-thu-muc-ragas-evaluation>

--------------------------------------------------------------------------------
2. PHUONG PHAP CHINH - RAGAS FILTER
--------------------------------------------------------------------------------

  Muc tieu: trich dac trung RAGAS (khong dung gold answer luc suy luan),
  roi huan luyen bo loc sklearn accept/reject.

  2.1. Trich dac trung (notebook 3.1)

      Mo notebooks/3.1_ragas-feature-extraction.ipynb
      Restart kernel, chay lan luot cac cell.

      - Input : data/labeled_merged.csv
      - Can   : OPENAI_API_KEY trong .env
      - Model : gpt-4o-mini + text-embedding-3-small (temperature=0)
      - Metric dung (black-box):
            faithfulness, answer_relevancy, context_relevancy
        (khong dung answer_correctness / answer_similarity)
      - Mac dinh chay N_TIMES=3 (3 lan trich cung cau hinh)

      Output:
        results/ragas_filter/merged/merged_ragas_features_1.csv
        results/ragas_filter/merged/merged_ragas_features_2.csv
        results/ragas_filter/merged/merged_ragas_features_3.csv
        results/ragas_filter/merged/merged_ragas_checkpoints_*.csv

      Luu y: buoc nay goi API OpenAI — ton thoi gian va chi phi.
      Nen thu tren subset truoc khi chay toan bo (~9800 mau).

  2.2. Huan luyen bo loc (notebook 3.2)

      Mo notebooks/3.2_filter-training.ipynb
      Restart kernel, chay lan luot cac cell.

      - Input : labeled_merged.csv + file features tu buoc 2.1
      - Mo hinh sklearn (LR, RF, GB, ExtraTrees, HistGB, tuy chon XGBoost)
      - train_test_split(test_size=0.2, random_state=42) tren hang feature

      Output:
        models/ragas_filter/*.joblib
        results/ragas_filter/merged/average_results.csv
        results/ragas_filter/merged/summary_classification_report.csv
        results/ragas_filter/ragas_filter_merged_summary.csv

4. BASELINE DeBERTa / NLI
--------------------------------------------------------------------------------

  Khong bat buoc OpenAI. Can GPU de train nhanh (CPU van duoc).

  4.1. Chay bang script (khuyen nghi)

      python scripts/run_deberta_nli_baseline.py ^
        --config configs/experiments/filter_training.yaml

      (Linux/macOS dung \ de xuong dong; PowerShell dung ` hoac viet 1 dong.)

  4.2. Chay bang notebook

      Mo notebooks/5_deberta_nli_baseline.ipynb
      Restart kernel -> Setup -> (tuy chon) gates -> Train

  Cau hinh:
    configs/experiments/filter_training.yaml
    configs/filtering/deberta_filter.yaml

  Mac dinh:
    - Train/val tu data/labeled_merged.csv (loai mau nam trong test)
    - Danh gia tren data/labeled_merged_test.csv (reuse_frozen_test: true)
    - n_runs: 1, seed: 42
    - Nguong tren val: min-FPR @ recall >= 0.70
    - Giu fp16: false trong deberta_filter.yaml

  Output:
    models/answer_filter/run_1/
    results/deberta_nli/run_1/
    results/deberta_nli/nli_zeroshot/
    results/deberta_nli/summary.json
    results/deberta_nli/summary_classification_report.csv

  Chi chay lai NLI (bo qua train DeBERTa):

      python scripts/run_deberta_nli_baseline.py ^
        --config configs/experiments/filter_training.yaml ^
        --skip-train --skip-overfit-gate

--------------------------------------------------------------------------------
5. DU LIEU VA SPLIT
--------------------------------------------------------------------------------

  data/labeled_merged.csv
      ~9800 mau; cot: id, question, context, answer, label, dataset
      Nguon: ASQA + MS MARCO + WikiEval
      Mau ao giac dung hau to _hallu (vi du asqa_0 / asqa_0_hallu)

  data/labeled_merged_test.csv
      Tap holdout dong bang (base-ID split, seed=42, test_size=0.2)
      Dung cho LLM-judge va DeBERTa / NLI

  Split DeBERTa: theo base question ID de cap (correct/_hallu) khong
  bi lo qua train/val/test. Cau hinh: configs/experiments/filter_training.yaml

  RAGAS notebook 3.2 dung train_test_split tren hang feature
  (khac protocol freeze test cua DeBERTa).

--------------------------------------------------------------------------------

  Moi bo loc nen bao cao:

    acceptance_rate, confusion matrix / FPR

    accuracy,precision,recall,f1,roc_auc
